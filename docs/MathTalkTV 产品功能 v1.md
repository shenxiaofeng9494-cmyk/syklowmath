# 一. 产品定义

## 1. 一句话定义

MathTalkTV是一个对话式视频教学平台，将传统数学教学视频升级为"可随时打断提问、实时获得AI教师回答"的互动体验。学生在观看视频的过程中遇到疑问，可以随时暂停并向AI提问，AI不仅能用教师的声音给出回答，还能调用画板推导公式、或在IDE中演示代码执行，讲解完毕后视频继续播放。

## 2. 核心痛点

当前在线数学学习的真实场景：

```
学生小明在看录播课视频
    ↓
老师说："接下来我们用配方法..."
    ↓
小明：？？？配方法是什么？
    ↓
现有解决方案：
  ❌ 回放视频（但老师还是那样讲，听不懂）
  ❌ 跳过继续看（后面更听不懂）
  ❌ 搜题软件（搜到的是题目答案，不是"这句话的意思"）
  ❌ 问老师/家长（不在身边 / 不好意思问）
    ↓
结果：小明在"概念理解"阶段就断裂了
```

对学生：在视频学习中卡住时，最需要的不是"更多视频"或"题目答案"，而是有人能即时解释"刚才那句话到底什么意思"。这个需求在线下课堂可以通过"举手提问"满足，但在线上录播课中暂时无法解决。

对老师：传统录播课程的规模化与个性化无法兼得。一位老师可以通过视频触达成千上万的学生，却无法及时回应每个学生的个性化疑问。

MathTalkTV通过AI教师分身技术打破这一困局，让学生在卡壳时能立刻获得帮助，而不是被迫暂停学习去寻找其他资源。与纯文字问答不同，MathTalkTV 的AI可以边说边画，在画板上推导公式、绘制函数图像、标注几何关系，或在IDE中演示代码执行过程，这种多模态讲解能力是产品的核心差异点。

## 3. 用户画像

核心用户：初中生（12-15岁）

人群特征：

- **学段：**7-9年级（初一到初三）
- **数学水平：**中等偏下（60-80分/100分）
- **学习场景：**课后在家自学，使用录播视频
- **主要困惑：**概念理解困难，而非不会做题

### 用户画像1：小明（初二学生）

基本信息：

- 年龄：14岁，初二
- 数学成绩：70分（班级中等）
- 学习习惯：认真但理解慢，不好意思问老师

痛点：看网课视频时经常卡在一句话上，反复回放还是听不懂，想问问题但老师不在线，搜题只能搜到答案，没人讲概念。

期待：能随时问"这句话什么意思"，立刻得到解释，不用等，且解释通俗易懂。

### 用户画像2：小红（初三学生）

基本信息：

- 年龄：15岁，初三
- 数学成绩：75分（中等偏上）
- 学习习惯：目标明确，想快速补弱项

痛点：某些概念一直搞不清楚（如配方法、函数），传统视频节奏不可控，想快速跳到困惑点，不想完整看一遍，只想针对性解决问题。

期待：能精准定位到自己不懂的知识点，只解释这一个点，不浪费时间，能反复问直到懂。

### 非核心用户（暂不考虑）

1. 小学生（理解力和自主学习能力不足）
2. 高中生（需求偏向解题技巧，而非概念理解）
3. 学霸（80分+，自学能力强，不需要反复解释）
4. 学困生（40分以下，需要一对一真人辅导）

## 4. 核心价值

### 1. 对学生

把"线下课堂随时举手提问"的体验，搬到线上视频学习中。不用因为看录播课时“卡住”而被迫停止学习。AI 就像坐在旁边的私教，随时可以在草稿纸（画板）上画图，或者在电脑（IDE）上演示计算，讲懂了就继续看课。

具体价值：

1. 即时性：不用等老师上线，AI立刻回答
2. 针对性：只解释"当前这句话"，不发散
3. 无压力：不用担心问题太简单被笑话
4. 可重复：同一个问题可以问多次，直到懂

### 2. 对家长

- 孩子自主学习能力提升（不用随时等着回答问题）
- 可视化学习过程（知道孩子在哪些知识点卡住）
- 性价比高（比一对一辅导便宜）

### 3. 对教培机构

- 降低教师答疑成本（重复性问题交给AI）
- 提升录播课价值（不再是"看完就没用"）且无需改变原有的录课习惯。
- 积累学生困惑数据（优化教学内容）
